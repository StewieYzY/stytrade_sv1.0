
# StGTrade AI v3.1.0 功能验证测试用例

## 测试用例 01：模型配置实时生效验证
**目的**：验证各智能体是否严格遵循“配置中心”分配的模型执行任务。
**步骤**：
1. 点击左侧导航栏的 **设置 (Settings)** 图标。
2. 将 `基金经理` 设置为 `Gemini 3 Pro (高精度)`。
3. 将 `舆情分析师` 设置为 `Gemini Lite (轻量速)`。
4. 返回 **分析 (Analysis)** 页面，输入代码 `NVDA` 并启动。
5. **预期结果**：
   - 在左侧“智能体状态”列表中，点击`基金经理`，其下方应显示金色的 `PRO` 标签。
   - 点击`舆情分析师`，其下方应显示蓝色的 `LITE` 标签。
   - 验证配置已持久化：刷新页面，进入设置，检查选项是否保持不变。

## 测试用例 02：全链路 PDF 报告导出顺序验证
**目的**：验证整合报告中“基金经理”的意见是否位于文档第一位。
**步骤**：
1. 进入 **历史决策档案 (History)**。
2. 如果没有记录，请前往 **设置** 使用“生成模拟历史数据”工具。
3. 点击任意一条历史记录进入详情页。
4. 点击右上角的 **导出决策报告 (PDF)**。
5. 在浏览器弹出的打印预览界面中查看内容。
6. **预期结果**：
   - 文档标题后，出现的第一个章节必须是 `[基金经理] 核心裁定`。
   - 检查页眉信息是否包含正确的股票名称、代码、基准价及 **系统版本 v3.1.0**。
   - 检查页面排版：每个智能体的报告应独立成段，且“基金经理”部分应有明显的蓝色视觉引导。

## 测试用例 03：配置中心编辑工作流验证
**目的**：验证编辑、锁定、取消逻辑。
**步骤**：
1. 进入 **设置 (Settings)**。
2. 尝试点击任意智能体的模型选择框。 (应为禁用状态)
3. 点击顶部 **编辑配置** 按钮。
4. 修改任意模型配置。
5. 点击 **取消**。
6. **预期结果**：配置恢复为原始状态，且选择框重新变为禁用。
